## Почему Claude Code – это не про код

Claude Code вышел на $2,5 млрд годовой выручки самостоятельно. За девять месяцев после запуска. Быстрее, чем ChatGPT, Slack или любой другой B2B-продукт в истории добрался до $1 млрд.

Все используют Claude Code как продвинутый автокомплит. Писать код быстрее. Дебажить быстрее. Шипить больше.

Это – деньги на столе, которые вы не забираете.

Настоящая игра – использовать Claude Code как слой оркестрации для бизнесов, не связанных с кодом. Пайплайны, которые вытягивают данные, обрабатывают их, генерируют результат и доставляют его клиенту.

Никаких приложений. Никакого SaaS. Никакой поддержки пользователей.

Только повторяемые рабочие процессы, которые решают дорогие проблемы.

## 1. Пайплайн нарезки видео

У каждого креатора, которого я знаю, одна и та же проблема.

50+ часов длинного видео лежат на YouTube. Опубликованы один раз. Больше никто к ним не возвращается.

Ручной процесс переупаковки выглядит так: посмотреть видео, промотать таймлайн, найти хорошие моменты, нарезать клипы, написать текст под каждую платформу, отформатировать, поставить в расписание.

Это 3–4 часа на одно видео. Каждую неделю. Навсегда.

Вот что я собрал вместо этого.

Вставляешь YouTube-ссылку в Claude Code (или любую ссылку на видео, или загружаешь файл напрямую).

Claude вызывает WayinVideo Clipping API. Получает 15+ клипов, ранжированных по показателю виральности (0–99). К каждому клипу прилагаются заголовок, описание, хештеги и ссылка на скачивание MP4 с CDN. Авторефрейминг до 9:16. Анимированные субтитры уже вшиты.

Параллельно – вызов Transcription API. Полная расшифровка с таймкодами и метками спикеров.

Дальше Claude фильтрует. Показатель выше 75. Длительность больше 20 секунд. Остается 7–10 клипов, которые стоит публиковать.

Для каждого клипа Claude извлекает соответствующий фрагмент расшифровки по совпадению таймкодов. Прогоняет его через промпт генерации копирайта с вашим голосовым профилем. На выходе – посты для LinkedIn, TikTok, Shorts, адаптированные под каждую платформу.

Еще один опциональный шаг: запушить все в CMS. Клип, текст, хук, первый комментарий, статус «На проверке».

Один YouTube-URL на входе. 10 готовых к публикации постов на выходе.

**API-вызовы, которые делают это возможным:**

- `POST /api/v2/clips` – ранжированные клипы со ссылками на экспорт
- `POST /api/v2/transcripts` – полная расшифровка с таймкодами
- `POST /api/v2/summaries` – карта контента для написания хуков
- `POST /api/v2/moments` – ручной выбор конкретных сегментов

Clipping API находит, что публиковать. Transcription API дает, о чем писать. Summarisation API строит карту. Find Moments API – это ручное управление, когда нужен конкретный фрагмент, который автоклиппер пропустил.

Я бы продавал это любому креатору или агентству, которое выпускает 2–3 видео в неделю.

**Математика:** 4 сэкономленных часа на видео × 3 видео в неделю × $50/час (ставка консалтинга) = $600/неделю сэкономленного времени. Берите $500/месяц за пайплайн. ROI продает сам себя.

## 2. Агент обогащения лидов

Команды продаж до сих пор делают это руками.

Найти потенциального клиента в LinkedIn. Открыть сайт компании. Загуглить. Скопировать данные в таблицу. Оценить по ощущениям. Вставить в CRM.

Это 8–12 минут на одного клиента. При 50 лидах в день – это полный рабочий день, потраченный на ввод данных, замаскированный под «поиск клиентов».

Claude Code решает это в четыре шага: загрузка, обогащение, скоринг, маршрутизация.

Один раз задаете критерии идеального клиента (ICP). Скармливаете Claude сырой список компаний или доменов. Claude заходит на сайт каждой компании, считывает индикаторы отрасли, размер команды, стек технологий, позиционирование, недавние упоминания в прессе. Оценивает каждый лид по шкале 0–100 на соответствие ICP. Подходящие записи попадают в HubSpot или чистую таблицу. Не подходящие – маркируются с указанием причины, чтобы команда перестала к ним возвращаться.

Слой обогащения – вот где становится интересно. Claude читает страницу «О нас» и извлекает, чем компания реально занимается (а не то, что написано в SEO-оптимизированном слогане). Проверяет вакансии на странице карьеры – из них можно вычислить стадию роста, приоритеты бюджета и технические решения. Если компания только что разместила три вакансии DevOps – она масштабирует инфраструктуру. Если появилась позиция Head of AI – она покупает, а не строит сама.

Сверху добавляется проход персонализации. Claude вытягивает свежий пост из блога или пресс-релиз каждого потенциального клиента и пишет однострочный оупенер, который SDR может вставить в первое письмо. Одна эта строка превращает холодное письмо в теплое.

**На выходе:** таблица с названием компании, ICP-скором, выжимкой из обогащения, сигналами покупки и персонализированным хуком. Готово к аутричу.

**Лучшие покупатели:** агентства, работающие с аутбаундом, рекрутинговые фирмы, B2B-команды продаж с 2–3 SDR, которые полдня тратят на ресерч вместо звонков.

Заметка про комплаенс. Если вы вытягиваете данные LinkedIn – используйте одобренные коннекторы или браузерные сценарии с данными только из публичных профилей. Позиционируйте это как обогащение из открытых источников. Никому не нужно письмо с требованием прекратить действия из-за таблицы для проспектинга.

## 3. Пайплайн конкурентной разведки

Продуктовые команды и маркетинг-лиды отслеживают конкурентов вручную. Кто-то каждый понедельник открывает пять вкладок, просматривает страницы с ценами, читает чейнджлоги, скриншотит таблицы фич и пишет в Slack саммари, которое прочитают три человека, прежде чем оно утонет в скролле.

Этот ритуал стоит 3–5 часов в неделю на одного аналитика. Масштабируется плохо. И пропускает изменения между проверками.

Claude Code заменяет утренний понедельничный ритуал аналитика автоматическим пайплайном.

Определяете список целей: сайты конкурентов, страницы с ценами, матрицы фич, страницы карьеры, пресс-румы. Claude скрейпит каждый из них по расписанию (ежедневно или еженедельно – как вам удобно). Сохраняет снэпшот. На следующем запуске сравнивает новую страницу с сохраненной версией и пишет человекочитаемое описание того, что изменилось.

*«Конкурент А поднял тариф Pro с $49 до $59. Конкурент Б добавил SOC 2 compliance на страницу enterprise. Конкурент В разместил 4 новых вакансии ML-инженеров за последнюю неделю.»*

Последний сигнал важнее, чем кажется. Паттерны найма раскрывают приоритеты роадмапа за шесть месяцев до того, как фича выйдет.

**Архитектура:** целевые URL хранятся в конфиг-файле. Акторы Apify или браузерная автоматизация Claude выполняют скрейпинг. Claude читает сырой HTML, извлекает структурированные данные, сравнивает с предыдущим снэпшотом в локальной SQLite-базе и отправляет отформатированный отчет в Slack или email.

Сверху можно наложить анализ цен. Отслеживать названия тарифов, комплектации фич и ценовые точки по 10 конкурентам в одной таблице. Когда кто-то меняет цены – ваша команда узнает в течение 24 часов.

**Лучшие покупатели:** продакт-менеджеры в B2B SaaS, маркетинговые команды, работающие над позиционированием, стратегические консультанты, ведущие нескольких клиентов в одной вертикали.

**Ценообразование:** $1 000–2 500 в месяц в зависимости от количества отслеживаемых конкурентов и частоты отчетов. Ваши вычислительные расходы – меньше $100/мес. Маржа – это слой аналитики, который вы строите сверху: знание того, какие сигналы важны, как интерпретировать паттерны найма, что означает изменение цен для позиционирования вашего клиента.

## 4. Пайплайн извлечения данных из документов

Скучно. Рутинно. Продается на ура.

Счета, чеки, договоры, заказы на закупку. В 2026 году все еще вбиваются в таблицы руками. Финансовые отделы в средних компаниях тратят 15–20 часов в неделю на ручной ввод данных из документов, которые следуют предсказуемым форматам.

Claude Code читает файл, извлекает нужные поля, валидирует по схеме и отправляет структурированные данные в таблицу или бухгалтерскую систему.

**Архитектура:** загрузить документ (PDF, скан, фото). Если это скан – прогнать OCR. Claude извлекает поля в фиксированную схему: название поставщика, номер счета, дата, строки позиций, итоги, налог, условия оплаты. Валидирует извлечение (сходится ли математика по строкам? совпадает ли номер PO с открытым заказом?). Чистые записи уходят в бухгалтерскую систему. Помеченные – в очередь на ревью человеком.

Продакшн-пайплайны показывают 94–97% точности на уровне полей для стандартных счетов. Строки позиций на грязных сканах все еще требуют ручной проверки. Эта честность – то, что делает продукт продаваемым. Вы не обещаете магию. Вы обещаете, что 95% рутины исчезнет, а оставшиеся 5% будут помечены с контекстом, чтобы человек разобрался за секунды, а не за минуты.

Claude Code добавляет слой, которого нет у большинства OCR-инструментов: понимание. Он читает договор и извлекает даты продления, условия расторжения, графики платежей. Читает чек и категоризирует расход. Извлечение – семантическое, а не позиционное. В этом разница между вытаскиванием текста по координатам на странице и пониманием того, о чем говорит документ.

Для компаний, обрабатывающих сотни документов в месяц, этот пайплайн экономит работу одного штатного сотрудника. Человек, который сейчас этим занимается, ненавидит эту работу. Его руководитель знает, что это риск для удержания персонала. Вы решаете обе проблемы.

**Лучшие покупатели:** финансовые отделы компаний на 50–500 сотрудников, бухгалтерские фирмы, операционные команды, обрабатывающие документы от поставщиков, юридические отделы, ревьюящие портфели договоров.

**Ценообразование:** по документу ($1–3 за счет) или фиксированная месячная ставка ($500–2 000) в зависимости от объема. Ваша себестоимость – центы на документ. Ценность каждого документа – 8–12 минут квалифицированного труда, которые он заменяет.

## 5. База знаний и автоматизация поддержки

У каждой SaaS-компании с более чем 1 000 пользователей одна и та же проблема: документация, которая отстает от продукта на полгода.

Инженеры шипят фичи. В чейнджлог попадает две строчки. Тикеты в поддержку начинают копиться: «как работает новая фича?». Технический писатель добавляет задачу в бэклог. Через три месяца статья публикуется. К тому моменту фича уже дважды изменилась.

Claude Code закрывает этот разрыв.

Пайплайн работает в два этапа.

**Первый – анализ пробелов.** Claude обходит вашу текущую документацию, вытягивает свежие тикеты из Zendesk или Intercom и сопоставляет. Определяет вопросы, которые клиенты задают регулярно, но на которые нет статьи в базе знаний. Ранжирует пробелы по объему тикетов: вопрос, заданный 200 раз за прошлый месяц и не имеющий статьи, получает приоритет над тем, что спросили 5 раз.

**Второй – генерация контента.** Claude пишет черновики статей для каждого пробела. Берет контекст из самих тикетов поддержки (реальные вопросы клиентов, реальные точки замешательства), из чейнджлога продукта и из существующих смежных статей. На выходе – формат, который уже использует ваша команда: заголовки, примеры кода, плейсхолдеры для скриншотов.

Черновики попадают в очередь на ревью. Технический писатель или продакт-менеджер утверждает, правит и публикует. Пайплайн запускается еженедельно. Каждый понедельник команда получает список новых черновиков, ранжированных по влиянию на клиентов.

Еще один уровень глубже: Claude может определять статьи, которые существуют, но устарели. Если тикеты ссылаются на фичу, а документация по этой фиче последний раз обновлялась восемь месяцев назад – Claude помечает ее для ревью и пишет обновленную версию на основе последних записей чейнджлога.

**Лучшие покупатели:** SaaS-компании с активными очередями поддержки, компании-разработчики инструментов с большими массивами документации, любой бизнес, где объем тикетов коррелирует с качеством документации.

**Ценообразование:** разовый сетап ($5 000–10 000 за подключение систем, настройку пайплайна и калибровку стиля) + ежемесячный ретейнер ($1 500–3 000) за регулярные запуски, мониторинг и тюнинг промптов.

**Математика ROI:** если ваша команда поддержки обрабатывает 2 000 тикетов в месяц и 30% из них имеют ответы, которые должны быть в документации – это 600 тикетов в месяц, которые можно отклонить за счет лучшей документации. При стоимости обработки тикета $5–15 – это $3 000–9 000/мес экономии на поддержке. Пайплайн окупается за первый месяц.

## Паттерн, общий для всех пяти

Ни один из них не требует создания приложения. Ни один не требует поддержки инфраструктуры. Ни один не требует технически подкованного покупателя.

Вы не продаете «AI-автоматизации». Вы упаковываете повторяющуюся бизнес-работу в пайплайны, которые клиенты уже понимают и за которые уже платят.

Claude Code – это слой оркестрации. API и инструменты – слой возможностей. Ваше знание проблемной области – это ваш конкурентный ров.

Выберите один пайплайн. Соберите его сначала для себя. Потом продайте человеку, у которого та же проблема, но в 10 раз больше масштаб. Это бизнес, который никто не строит.
