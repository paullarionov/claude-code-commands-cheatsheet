## 1. Мобильное приложение Claude Code

У Claude Code есть полноценное мобильное приложение — можно писать код прямо с телефона.

- Скачайте приложение Claude для iOS или Android
- Перейдите на вкладку **Code** в боковом меню
- Можно просматривать изменения, одобрять PR и писать код на ходу

---

## 2. Перенос сессий между устройствами

Команда `claude --teleport` (или `/teleport`) позволяет продолжить облачную сессию на локальной машине. А `/remote-control` даёт управление локальной сессией с телефона или из браузера.

- **Teleport** — перетягивает облачную сессию в локальный терминал
- **Remote Control** — управление локальной сессией с любого устройства
- Совет: включите «Enable Remote Control for all sessions» в `/config`, чтобы не настраивать каждый раз

## 3. /loop и /schedule — автоматизация по расписанию

Эти команды запускают Claude автоматически с заданным интервалом — до недели непрерывной работы. Примеры:

- `/loop 5m /babysit` — каждые 5 минут проверяет код-ревью, делает rebase и проводит PR до продакшена
- `/loop 30m /slack-feedback` — каждые 30 минут создаёт PR по фидбеку из Slack
- `/loop /post-merge-sweeper` — создаёт PR для пропущенных замечаний код-ревью
- `/loop 1h /pr-pruner` — закрывает устаревшие и ненужные PR

Идея простая: превратите повторяющееся действие в навык (skill), а потом поставьте его на цикл (loop).

## 4. Хуки: автоматические действия при событиях

Хуки — это скрипты, которые автоматически запускаются, когда Claude делает определённое действие. Работают как триггеры: «когда произошло X — выполни Y». В отличие от инструкций в CLAUDE.md, хуки срабатывают **всегда**, без исключений.

Примеры:

- **При старте сессии** (`SessionStart`) — подгрузить нужный контекст из файлов или API
- **Перед выполнением команды** (`PreToolUse`) — залогировать каждую bash-команду, которую Claude собирается запустить
- **При запросе разрешения** (`PermissionRequest`) — отправить уведомление в WhatsApp, чтобы одобрить или отклонить действие с телефона
- **При остановке** (`Stop`) — заставить Claude продолжить работу, если задача ещё не закончена

## 5. Cowork Dispatch — удалённое управление десктопным Claude

Dispatch позволяет разбирать Slack, почту, управлять файлами и делать дела на ноутбуке, даже когда вы не за компьютером.

- Dispatch — это **безопасное удалённое управление** десктопным приложением Claude
- Он может использовать ваши MCP-серверы, браузер и компьютер — с вашего разрешения
- По сути, способ делегировать не-кодовые задачи Claude откуда угодно

## 6. Chrome-расширение для фронтенд-работы

Главный совет: **дайте Claude возможность увидеть результат.** Когда он может проверить, как выглядит страница, он будет дорабатывать код, пока результат не станет хорошим.

- Аналогия: вы просите человека сделать сайт, но запрещаете открывать браузер — результат вряд ли будет хорошим
- Дайте Claude браузер — и он начнёт итерировать до нужного качества
- Chrome-расширение работает надёжнее аналогичных MCP-серверов для этой задачи

## 7. Десктопное приложение: автозапуск и тестирование веб-серверов

Десктопное приложение Claude умеет **автоматически поднимать веб-сервер и тестировать его во встроенном браузере.**

- Аналогичную настройку можно сделать в CLI или VS Code через Chrome-расширение
- Но десктопное приложение даёт это из коробки, без дополнительной настройки

## 8. Ветвление сессии

Иногда посреди разговора хочется «свернуть в сторону» — попробовать другой подход, не теряя основную сессию. Два способа:

1. Выполните `/branch` прямо в сессии
2. Из терминала: `claude --resume <session-id> --fork-session`

`/branch` создаёт ответвление — вы переключаетесь в него. Чтобы вернуться к оригиналу: `claude -r <original-session-id>`.

---

## 9. /btw — побочные вопросы без прерывания работы

Claude выполняет задачу, а вам нужно быстро что-то уточнить? `/btw` позволяет задать вопрос, не прерывая текущую работу агента.

Пример:

```
/btw как пишется дакель«»?
> Такса — от немецкого Dachshund: Dachs барсук() + Hund собака().
```

Агент продолжает работать как ни в чём не бывало.

## 10. Git Worktrees — параллельная работа в одном репозитории

В Claude Code встроена глубокая поддержка git worktrees. Это позволяет запустить **десятки параллельных сессий Claude** в одном репозитории, и каждая будет работать в изолированной копии.

- `claude -w` — запуск новой сессии в отдельном worktree
- Или отметьте чекбокс **worktree** в десктопном приложении
- Для систем контроля версий, отличных от git, используйте хук `WorktreeCreate` со своей логикой

---

## 11. /batch — массовые изменения через параллельные агенты

`/batch` сначала уточняет задачу, а затем распределяет работу между **агентами в отдельных worktrees** — десятки, сотни, даже тысячи одновременно.

- Подходит для масштабных миграций кода и другой параллелизуемой работы
- Каждый агент работает независимо в своей копии репозитория

---

## 12. –bare — ускорение запуска SDK до 10x

По умолчанию `claude -p` (и TypeScript/Python SDK) при запуске ищет локальные CLAUDE.md, настройки и MCP-серверы. Для автоматизации и скриптов это лишнее — обычно вы сами указываете, что загружать, через `--system-prompt`, `--mcp-config`, `--settings`.

- Флаг `--bare` пропускает автопоиск и **ускоряет запуск до 10 раз**
- В будущих версиях это станет поведением по умолчанию

```bash
claude -p "summarize this codebase" \
    --output-format=stream-json \
    --verbose \
    --bare
```

## 13. –add-dir — работа с несколькими репозиториями

Если нужно работать сразу в двух репозиториях — запустите Claude в одном, а через `--add-dir` (или `/add-dir`) подключите второй.

- Это не просто показывает Claude другой репозиторий — оно **даёт права на работу** в нём
- Для постоянной настройки добавьте `"additionalDirectories"` в `settings.json` проекта

---

## 14. –agent — свой агент с ограниченными правами

Можно создать специализированного агента с собственным системным промптом и набором инструментов. Определите его в `.claude/agents/` и запустите:

```bash
claude --agent=<имя агента>
```

- У агента могут быть ограниченные инструменты (например, только чтение), своя модель и описание
- Удобно для read-only ревьюеров, аудиторов безопасности, доменных помощников

---

## 15. /voice — голосовой ввод

Можно писать код голосом, а не набирая текст.

- В CLI: выполните `/voice`, затем зажмите пробел для записи
- В десктопном приложении: нажмите кнопку голосового ввода
- Или включите системную диктовку в настройках iOS
