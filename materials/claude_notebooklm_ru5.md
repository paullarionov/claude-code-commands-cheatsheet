## Проблема: лимиты токенов и амнезия Claude

Знакомо сообщение «Claude usage limit reached. Your limit will reset at 7pm»? Если да — эта статья для вас.

Проблема в том, что амнезия Claude сжирает токены. На плане Pro ($20/мес) лимиты заканчиваются быстро. На Max ($100-200/мес) запас больше, но тяжелое исследование все равно съедает бюджет.

Хотите, чтобы Claude проанализировал 30 документов, нашел связи между ними и написал отчет? Это дорогой день.

Решение — мост между Claude Code и Google NotebookLM. Тяжелую аналитическую работу перекладываем на бесплатную инфраструктуру Google, а токены Claude тратим только на оркестрацию и финальную доработку.

---

## Что такое Claude Code (коротко)

Если вы пользовались Claude только через чат — Claude Code это совсем другой зверь.

Он работает в терминале. Читает весь ваш код. Пишет файлы, запускает скрипты, параллельно поднимает субагентов, выполняет многоэтапные процессы без ручного контроля на каждом шаге.

Проблема — модель тарификации. Каждый фрагмент контекста, который вы скармливаете Claude, сжигает токены. На Pro лимиты заканчиваются быстро. На Max побольше, но интенсивная исследовательская работа все равно выжимает бюджет досуха.

---

## Что дает NotebookLM

NotebookLM — исследовательский инструмент от Google на базе RAG (retrieval-augmented generation). Вы загружаете документы, он все индексирует и позволяет задавать вопросы по всем источникам сразу.

Цифры:

- Бесплатный тариф: 50 источников на блокнот
- Pro-тариф: 300 источников на блокнот
- Стоимость обработки: ноль

Поддерживает PDF, веб-ссылки, видео с YouTube, Google Docs, текст, аудио, изображения. Поскольку ответы привязаны к загруженным источникам, он не галлюцинирует так, как универсальный чатбот.

Ограничение: нет официального API. Это инструмент только для браузера. Нельзя ни заскриптовать, ни автоматизировать, ни подключить к чему-то.

Именно эту проблему решает **notebooklm-py**.

## Установка моста

Разработчик Тенг Линг (Teng Ling) реверс-инжинирил внутренние протоколы NotebookLM и опубликовал open-source CLI-инструмент **notebooklm-py**. Он позволяет управлять NotebookLM целиком из терминала: создавать блокноты, загружать источники, выполнять запросы, генерировать слайды, подкасты, карточки.

### Что нужно

- Python 3.10+
- Аккаунт Google
- Терминал (macOS, Linux, Windows)

### Установка

Репозиторий: https://github.com/teng-lin/notebooklm-py

Следуйте инструкциям в README.

## Обучаем Claude Code работать с NotebookLM

Мост установлен. Теперь нужно дать Claude знание о том, как его использовать. Для этого используются **навыки (skills)**.

### Навыки за 30 секунд

Навык — это набор инструкций в файле SKILL.md, который Claude читает при обнаружении подходящей задачи. Считайте его плейбуком на вашей машине. Claude загружает его автоматически, когда распознает совпадение с запросом.

Навыки следуют открытому стандарту. Работают в Claude Code, Cursor, Gemini CLI, Codex и других. Без привязки к вендору.

### Где хранятся навыки

**Персональные** (доступны во всех проектах):

```
~/.claude/skills/skill-name/SKILL.md
```

**Проектные** (привязаны к одному репозиторию, можно шарить через Git):

```
.claude/skills/skill-name/SKILL.md
```

Проектные навыки перезаписывают персональные, если совпадают.

### Установка навыка NotebookLM

```
notebooklm skill install
```

Навык устанавливается в два места: `~/.claude/skills/notebooklm/` (для Claude Code) и `~/.agents/skills/notebooklm/` (для совместимых агентов вроде Codex).

Проверяем:

```
notebooklm skill status
```

После установки Claude понимает, как создавать блокноты, загружать источники, выполнять запросы и генерировать результаты через CLI. Не нужно объяснять синтаксис каждую сессию.

### Как Claude решает, когда использовать навык

У каждого навыка есть описание в заголовке. Claude читает все доступные описания при запуске и сопоставляет их с вашим запросом. Попросите «исследуй стратегии B2B-продаж и собери отчет» — и он подтянет навык NotebookLM автоматически.

Можно вызвать и напрямую: `/notebooklm`

### Создание своих навыков

Anthropic публикует мета-навык **skill-creator**. Вызовите `/skill-creator` в Claude Code — он проведет интервью о том, что вы хотите, сгенерирует полный SKILL.md, прогонит тестовые промпты и упакует результат.

От идеи «хочу навык, который делает X» до работающего, протестированного навыка — минуты, не часы.

## Четыре рабочих процесса

Установка — скучная часть. Ценность — в этих четырех процессах.

### Процесс А: Исследование без расхода токенов

**Проблема:** нужно проанализировать 30+ документов. Делать это локально — убить бюджет токенов.

**Решение:** Claude оркестрирует, NotebookLM обрабатывает. Бесплатно.

#### Пошагово

**1. Соберите источники.** PDF, статьи, транскрипции YouTube (для извлечения транскрипций Claude может использовать yt-dlp).

**2. Claude создает блокнот:**

```
notebooklm create "My Research Project"
```

**3. Claude загружает все источники:**

```
notebooklm source add \
  "./transcript-1.md" \
  "https://example.com/article" \
  "./report.pdf"
```

До 50 источников на бесплатном тарифе — для большинства проектов хватает.

**4. Claude делает запросы к NotebookLM вместо локальной обработки:**

```
notebooklm ask \
  "what are the three most important themes \
  across all sources?"
```

Движок Gemini от Google обрабатывает запрос по всем загруженным документам. Возвращает ответ с цитатами. Бесплатно.

**5. Claude генерирует результаты:**

```
notebooklm generate slide-deck
notebooklm generate flashcards --quantity more
notebooklm generate mind-map
notebooklm generate data-table "compare key concepts"
notebooklm generate audio "make it engaging" --wait
```

Все скачивается на вашу машину.

**6. Claude полирует результат.** Берет сырые артефакты и дорабатывает локально: редактирует слайды, переформатирует таблицы, интегрирует находки в финальный документ. Только этот шаг расходует ваши токены Claude.

#### Арифметика

Дорогая аналитическая работа происходит на инфраструктуре Google. Токены Claude зарезервированы исключительно для оркестрации и финальной редактуры. Вы только что растянули план за $20/мес до возможностей процесса за $200/мес.

### Процесс В: Создание экспертных ИИ-агентов из веб-исследований

Вы хотите собрать кастомного ИИ-агента для конкретной области (например, B2B-продажи). Но расплывчатые промпты дают расплывчатых агентов.

Используйте Deep Research от NotebookLM для автономного сбора экспертных знаний из интернета, а затем структурируйте их в навык для Claude Code.

#### Пошагово

**1. Запустите Deep Research в NotebookLM.** Откройте NotebookLM в браузере, выберите тип источника «web» и введите конкретный запрос:

«advanced B2B multi-channel outbound sales strategies, retention loops, and re-engagement sequences»

Deep Research автономно обходит сотни страниц, читает документацию и гайды, собирает отчет с цитатами.

**2. Структурируйте результат по фреймворку DBS:**

- **Direction** (направление) — пошаговая логика, деревья решений, обработка ошибок. Становится ядром SKILL.md.
- **Blueprints** (чертежи) — статические справочные материалы: шаблоны, гайдлайны по тону, правила классификации. Становятся сопроводительными файлами.
- **Solutions** (решения) — задачи, требующие детерминированного кода, а не рассуждений ИИ: вызовы API, форматирование данных, расчеты. Становятся скриптами.

**3. Скормите результат skill-creator.** Скопируйте DBS-результат, вставьте в Claude Code и вызовите `/skill-creator`. Claude автоматически создаст весь пакет навыка.

**4. Протестируйте и разверните.** Skill-creator стресс-тестирует новый навык сгенерированными промптами, показывает результаты и позволяет дорабатывать до нужного уровня.

#### Результат

От расплывчатой идеи до работающего экспертного ИИ-агента, подкрепленного проверенными веб-исследованиями. Не за часы — за минуты.

### Процесс С: Память между сессиями

Вы потратили три часа, обучая Claude вашим архитектурным предпочтениям, конвенциям именования, особенностям проекта. Закрыли терминал. Все пропало.

Решение: ритуал «wrap-up», который автоматически извлекает наработки сессии и сохраняет их в персистентный блокнот NotebookLM. Claude обращается к нему в начале каждой следующей сессии.

#### Пошагово

**1. Установите навык `/wrap-up**, который инструктирует Claude просмотреть текущую сессию и извлечь:

- Ваши исправления (где Claude ошибся)
- Удачные паттерны, которые сработали
- Нерешенные вопросы и запросы на фичи
- Ключевые решения и их обоснования

**2. Настройте выгрузку в NotebookLM.** Вместо сохранения в локальный файл, wrap-up пушит итоги в выделенный блокнот «Master Brain»:

```
notebooklm use master-brain-notebook-id \
  "./session-summary-2026-04-06.md"
```

**3. Запускайте `/wrap-up` перед закрытием каждой сессии.** Claude просматривает разговор, извлекает инсайты, форматирует и выгружает.

**4. Добавьте инструкцию в CLAUDE.md.** Это конфиг, который Claude читает при запуске сессии. Добавьте:

«Прежде чем отвечать на вопросы об архитектуре проекта, исторических решениях или моих предпочтениях, запроси блокнот Master Brain через NotebookLM CLI.»

**5. У Claude теперь есть память.** За недели Master Brain накапливает сотни итогов сессий. NotebookLM индексирует их все и строит семантические связи. Claude извлекает именно тот контекст, который нужен, не загружая сотни документов в свое контекстное окно.

Ваш ИИ-агент реально запоминает все. Хранение и поиск происходят на бесплатной инфраструктуре Google. Бюджет токенов не страдает.

### Процесс D: Визуальное управление знаниями через Obsidian

Claude генерирует исследовательские документы, итоги сессий, аналитические файлы. Они копятся невидимыми файлами в директориях терминала. Вы не можете их просматривать, искать или связывать.

Решение: запустить Claude Code из корня хранилища Obsidian, чтобы все, что он создает, сразу отображалось в визуальном графе знаний.

#### Что такое Obsidian

Бесплатное приложение для заметок, работающее на локальных Markdown-файлах. Отображает заметки как интерактивный граф со связанными узлами. Очень популярно для управления личными знаниями.

#### Пошагово

**1. Запустите Claude Code из корня хранилища:**

```bash
cd ~/Documents/MyVault
claude
```

Claude получает полный доступ на чтение и запись ко всей коллекции заметок.

**2. Создайте CLAUDE.md в корне хранилища.** Это инструкция для Claude по работе с вашим хранилищем:

- Структура папок (куда идут исследовательские заметки, куда логи)
- Обязательные метаданные для новых заметок (даты, теги, ссылки на источники)
- Правила ссылок (оборачивать значимые концепции в двойные скобки `[[вот так]]` для графа Obsidian)
- Стандарты форматирования

**3. Создайте кастомные навыки для работы с хранилищем:**

- `/research <тема>` — Claude запрашивает NotebookLM, скачивает результаты, создает заметку с метаданными и перекрестными ссылками
- `/daily` — генерирует ежедневную сводку со ссылками на все, над чем работали
- `/wrap-up` — навык памяти из Процесса С, сохраняющий прямо в хранилище

**4. Дорабатывайте в реальном времени.** По мере того как Claude создает файлы, вы видите их появление в Obsidian. Если он неправильно категоризировал заметку или упустил перекрестную ссылку — поправьте и попросите Claude обновить CLAUDE.md. Цикл обратной связи обучает Claude соответствовать вашим предпочтениям.

#### Результат

Живая, растущая база знаний. Claude сортирует, тегирует и связывает информацию по вашим спецификациям. NotebookLM обрабатывает тяжелые исследования в фоне. Вы видите все в графе Obsidian.

## Что может пойти не так

Эта связка мощная. Но она построена поверх неофициального инструмента, который может сломаться завтра.

### Неофициальный API — нет гарантий

notebooklm-py реверс-инжинирит внутренние протоколы Google. Google это не одобряет. Если Google поменяет бэкенд, команды перестанут работать. Мейнтейнер (Тенг Линг) пока отзывчив, но никаких SLA нет.

Относитесь к этому как к инструменту для продвинутых пользователей, а не как к продакшен-инфраструктуре.

### Политика использования Anthropic

Anthropic требует, чтобы автоматизированные процессы использовали официальный клиент Claude Code с соответствующим тарифным планом. Не используйте эту связку для обхода лимитов токенов через неофициальные обвязки. Убедитесь, что ваше использование соответствует вашему плану.

### Данные и приватность (Великобритания, ЕС)

Если вы работаете с конфиденциальными или регулируемыми данными: пользовательские инструменты Claude обрабатывают и хранят данные в США. Последствия для GDPR реальны. Корпоративный API предлагает региональную обработку, но пользовательский тариф — нет.

### Защитите файл с куками

Файл `storage_state.json` содержит живые сессионные куки Google. Любой, кто получит этот файл, получит доступ к вашим данным NotebookLM.

- Никогда не коммитьте его в публичный репозиторий
- Обращайтесь с ним как с паролем

### Куки истекают

Периодически придется повторно авторизоваться. Если команды начинают падать с ошибками авторизации:

```
notebooklm login
```

Занимает 30 секунд.

## Краткий справочник команд

| Команда | Описание |
|---------|----------|
| `notebooklm create "Name"` | Создать блокнот |
| `notebooklm source add file url ...` | Добавить источники |
| `notebooklm ask "question"` | Запрос к блокноту |
| `notebooklm generate slide-deck` | Сгенерировать слайды |
| `notebooklm generate flashcards` | Сгенерировать карточки |
| `notebooklm generate mind-map` | Сгенерировать карту |
| `notebooklm generate audio` | Сгенерировать аудио |
| `notebooklm generate data-table "..."` | Сгенерировать таблицу |
| `notebooklm skill install` | Установить навык |
| `notebooklm skill status` | Проверить статус навыка |
| `notebooklm login` | Повторная авторизация |

## Что изучить дальше

**Соберите библиотеку навыков.** Каждый повторяющийся процесс — кандидат на навык. Используйте skill-creator, чтобы упаковать процессы один раз и переиспользовать.

**Изучите экосистему навыков.** Сообщество опубликовало тысячи навыков в официальном репозитории Anthropic на GitHub и на маркетплейсах вроде SkillsMP: код-ревью, генерация контента, пайплайны деплоя.

**Комбинируйте с MCP-серверами.** Model Context Protocol позволяет Claude Code общаться с внешними сервисами — GitHub, Slack, базы данных. Наложите это на NotebookLM для исследований и навыки для управления процессами — и получите полноценный автономный движок.

**Добавьте плагины Obsidian.** Dataview для динамических запросов по заметкам. Templater для автоматических шаблонов заметок. В сочетании с генерацией файлов через Claude хранилище становится чем-то ближе ко второму мозгу, чем к приложению для заметок.
