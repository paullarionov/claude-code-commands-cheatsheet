# Руководство по Prompt Engineering

Перевод для ТГ-канала @prompt_design

## Часть I. Введение в Prompt Engineering

### Что такое Prompt Engineering

Prompt Engineering — практика разработки и оптимизации входных данных (промптов) для языковых моделей с целью получения желаемых выходных данных.

Хороший промпт не просто задаёт вопрос — он управляет поведением модели: задаёт роль, контекст, формат и ограничения.

### Почему это важно

LLM чувствительны к формулировкам. Одна и та же задача, заданная по-разному, даёт кардинально разные результаты. Понимание техник промптинга позволяет:

- Получать более точные и релевантные ответы
- Снижать количество галлюцинаций
- Управлять форматом вывода
- Решать сложные многошаговые задачи

### Анатомия промпта

Хорошо структурированный промпт обычно включает:

- **Инструкция** — что нужно сделать
- **Контекст** — дополнительная информация для выполнения задачи
- **Входные данные** — конкретный вопрос или задача
- **Индикатор вывода** — формат или тип ожидаемого ответа

Не все промпты требуют всех четырёх компонентов. Их наличие зависит от задачи.

## Основные принципы

### Конкретность

Чем точнее формулировка, тем точнее результат.

**Плохо:** «Напиши что-нибудь про маркетинг»

**Хорошо:** «Напиши три идеи для email-кампании для SaaS-продукта, ориентированного на малый бизнес. Каждая идея — 2-3 предложения»

### Ролевые инструкции

Назначение роли улучшает качество ответов в специализированных областях:

```
You are an expert marketing copywriter with 10 years of experience
in B2B SaaS. Write a compelling product description for...
```

### Форматирование вывода

Явно указывайте желаемый формат:

- «Ответь в формате JSON с полями name, description, price»
- «Структурируй ответ как маркированный список»
- «Используй таблицу для сравнения»

## Примеры промптов

### Базовый промпт

```
Классифицируй отзыв клиента как позитивный, нейтральный или негативный.

Отзыв: "Доставка заняла 2 недели вместо обещанных 3 дней, но сам
товар отличного качества."
```

### Промпт с контекстом

```
Ты — ассистент службы поддержки интернет-магазина электроники.
Клиент пишет:

"Мой ноутбук не заряжается уже второй день. Что делать?"

Дай структурированный ответ с 3-4 шагами диагностики.
```

## Параметры генерации

### Temperature

Контролирует случайность вывода:
- **0.0–0.3**: детерминированные, предсказуемые ответы. Для фактических вопросов, кода, классификации.
- **0.7–1.0**: более творческие, разнообразные ответы. Для брейнсторминга, креативного письма.

### Top-p (nucleus sampling)

Ограничивает выборку токенами, суммарная вероятность которых не превышает p. `top_p=0.9` — модель выбирает из токенов, составляющих 90% вероятности.

### Max tokens

Ограничение длины вывода. Устанавливайте разумные пределы, чтобы избежать обрезания важных частей ответа.

## Часть II. Техники промптинга

### Zero-Shot Prompting

Модель выполняет задачу без примеров.

```
Classify the sentiment of the following movie review:
Review: "The movie was okay, nothing special but not terrible either."
Sentiment:
```

Работает хорошо для задач, которые модель «видела» в обучающих данных. Для специфических форматов или нюансированных задач нужны примеры.

---

### Few-Shot Prompting

Добавление нескольких примеров (shots) резко улучшает качество.

```
Classify the sentiment.

Review: "This is awesome!"
Sentiment: Positive

Review: "This is bad!"
Sentiment: Negative

Review: "The movie was okay."
Sentiment: Neutral

Review: "The food here is exceptional!"
Sentiment:
```

**Советы по Few-Shot:**
- Примеры должны быть разнообразными и репрезентативными
- Порядок примеров влияет на результат (недавние примеры весят больше)
- Оптимальное число примеров: 3–8 для большинства задач

---

### Chain-of-Thought (CoT) Prompting

Побуждает модель «думать вслух» перед ответом. Критично для задач, требующих рассуждений.

**Без CoT:**
```
Q: Roger has 5 tennis balls. He buys 2 more cans of tennis balls.
Each can has 3 balls. How many balls does he have now?
A: 11
```

**С CoT:**
```
Q: Roger has 5 tennis balls. He buys 2 more cans of tennis balls.
Each can has 3 balls. How many balls does he have now?
A: Roger started with 5 balls. 2 cans × 3 balls = 6 new balls.
5 + 6 = 11. The answer is 11.
```

**Zero-Shot CoT** — просто добавить «Let's think step by step»:

```
Q: A juggler can juggle 16 balls. Half of the balls are golf balls,
and half of the golf balls are blue. How many blue golf balls are there?
A: Let's think step by step.
```

Эта простая фраза значительно улучшает результаты на математических и логических задачах.

---

### Self-Consistency

Запускайте один и тот же CoT-промпт несколько раз с `temperature > 0`, затем берите большинство голосов.

Повышает точность на сложных задачах рассуждения. Дорого по токенам, но оправдано для критичных решений.

---

### Generated Knowledge Prompting

Сначала генерируйте знания, потом используйте их для ответа.

**Шаг 1:** «Сгенерируй 5 фактов о [теме]»

**Шаг 2:** Используйте эти факты как контекст для следующего промпта

Особенно полезно для вопросов о специфических доменах или для снижения галлюцинаций.

---

### Tree of Thoughts (ToT)

Расширение CoT — модель исследует несколько путей рассуждения параллельно.

Структура:
1. Разбить задачу на промежуточные «мысли»
2. Генерировать несколько вариантов на каждом шаге
3. Оценивать перспективность каждой ветки
4. Продолжать наиболее перспективные

Полезно для задач планирования, написания кода со сложными зависимостями, стратегических решений.

---

### RAG — Retrieval Augmented Generation

Объединяет поиск по базе знаний с генерацией. Модель получает релевантный контекст из внешних источников.

Схема:
```
Запрос пользователя → Поиск в векторной БД → 
Топ-K документов → Промпт с контекстом → Ответ
```

Преимущества:
- Актуальные данные без дообучения
- Ссылки на источники
- Контроль знаний без переобучения

---

### ReAct (Reasoning + Acting)

Чередует рассуждение (Thought) и действие (Action):

```
Question: What is the elevation range for the area that the
eastern sector of the Colorado orogeny extends into?

Thought: I need to search Colorado orogeny...
Action: Search[Colorado orogeny]
Observation: [результат поиска]

Thought: The eastern sector extends into High Plains. Need elevation.
Action: Search[High Plains]
Observation: [результат поиска]

Thought: High Plains range from 1,800 to 7,000 ft.
Action: Finish[1,800 to 7,000 ft]
```

ReAct снижает галлюцинации, комбинируя рассуждение с внешними инструментами.

---

### Reflexion

Агент рефлексирует над собственными ошибками и улучшает подход:

1. Агент пытается решить задачу
2. Получает обратную связь (ошибку или оценку)
3. Записывает «урок» в вербальную память
4. Повторяет попытку с учётом рефлексии

Значительно улучшает результаты в многошаговых задачах кодирования и рассуждения.

---

### Multimodal CoT

Расширение CoT для задач с изображениями:

1. **Фаза рассуждения**: модель анализирует изображение и текст, генерирует цепочку рассуждений
2. **Фаза вывода**: используя рассуждения, выдаёт финальный ответ

Модели 1B параметров с Multimodal CoT превосходят GPT-3.5 (175B) на некоторых бенчмарках.

---

### Graph Prompting

Представление задачи в виде графа для структурированного рассуждения. Полезно для задач с явными отношениями между сущностями.

---

### Active-Prompt

Адаптирует Few-Shot примеры к конкретной задаче:

1. Запустить модель на задачах без CoT-примеров
2. Выбрать задачи с наибольшей неопределённостью (высокое расхождение между ответами)
3. Попросить аннотаторов создать CoT для этих задач
4. Использовать аннотированные примеры для дальнейшего обучения

---

### Directional Stimulus Prompting

Добавление «стимула» — подсказки или ключевого слова, направляющего генерацию:

```
Summarize the article. Focus on: climate impact, economic costs,
policy recommendations.

Article: [текст статьи]
```

---

### Automatic Prompt Engineer (APE)

LLM генерирует и оценивает промпты автоматически:

1. Сгенерировать кандидатные промпты
2. Оценить каждый на валидационном наборе
3. Выбрать лучший по метрике качества
4. Итеративно улучшать

---

### PAL — Program-Aided Language Models

Модель генерирует программу (Python) для решения задачи вместо текстового ответа:

```python
# Q: I have a garden with 5 rows and 8 columns of plants.
# After removing 3 diseased plants, how many remain?
rows = 5
columns = 8
total = rows * columns  # 40
removed = 3
remaining = total - removed  # 37
print(remaining)
```

Устраняет ошибки вычислений, передавая математику интерпретатору.

---

### Prompt Chaining

Разбивка сложной задачи на последовательность промптов, где выход одного — вход следующего.

Преимущества:
- Лучшая производительность на сложных задачах
- Прозрачность и управляемость LLM-приложений
- Простая отладка

Пример — QA по документу в два шага:

1. Промпт 1: извлечь релевантные цитаты из документа
2. Промпт 2: сформировать ответ на основе цитат и документа

## Часть III. Применения LLM

### Генерация кода

LLM (например, ChatGPT) эффективны в генерации кода. Системное сообщение задаёт поведение:

```
You are a helpful code assistant that can teach a junior developer how to
code.
Your language of choice is Python. Don't explain the code, just generate the
code block itself.
```

Возможности:
- **Генерация из комментариев** — модель создаёт код по описанию в комментариях
- **Автодополнение функций** — модель завершает частично написанные функции
- **Генерация SQL-запросов** — модель генерирует запросы по описанию схемы базы данных
- **Объяснение кода** — модель разъясняет логику существующего кода

Пример генерации SQL:

```
"""
Table departments, columns = [DepartmentId, DepartmentName]
Table students, columns = [DepartmentId, StudentId, StudentName]
Create a MySQL query for all students in the Computer Science department
"""
```

Модель генерирует корректный запрос с `INNER JOIN`. Далее можно попросить создать схему БД и тестовые данные.

---

### Вызов функций (Function Calling)

Вызов функций — возможность надёжного подключения LLM к внешним инструментам и API.

Модели вроде GPT-4 обучены определять, когда нужно вызвать функцию, и генерировать JSON с аргументами для вызова. Это позволяет создавать:

- Диалоговых агентов, использующих внешние API
- Решения для извлечения и разметки данных
- Приложения, конвертирующие естественный язык в API-запросы
- Диалоговые системы поиска знаний

Пример: запрос «What is the weather like in London?» → модель возвращает вызов функции `get_current_weather(location="London", unit="celsius")` → разработчик вызывает реальный API → ответ передаётся обратно модели для финального ответа.

---

### Генерация данных

LLM способны генерировать согласованные данные для экспериментов и оценки:

Промпт:
```
Produce 10 exemplars for sentiment analysis. Examples are categorized as either
positive or negative. Produce 2 negative examples and 8 positive examples.
Use this format for the examples:
Q: <sentence>
A: <sentiment>
```

Модель генерирует 10 корректных примеров с правильным соотношением положительных и отрицательных.

### Разнообразие синтетических датасетов

Eldan et al. (2023) работали над генерацией детских рассказов. Основная проблема — обеспечение разнообразия. Решение:

1. Подготовка словаря (~1 500 базовых слов: существительные, глаголы, прилагательные)
2. Случайный выбор одного глагола, существительного и прилагательного для каждой генерации
3. Случайный выбор характеристик рассказа (диалог, неожиданный поворот, мораль)

Метод значительно расширяет разнообразие датасета.

**Textbooks Are All You Need**: Gunasekar et al. (2023) показали, что обучение на синтетических данных, напоминающих учебники (ясных, информативных, непредвзятых), существенно повышает качество. Модель Phi-1 (1,5B параметров) конкурирует с моделями в 10 раз крупнее.

Алгоритм для своих задач:

1. Определить переменные параметры между сэмплами
2. Подготовить коллекцию сущностей для заполнения
3. Генерировать данные с случайными подстановками (temperature выше default, ниже максимума)
4. Обучить локальную модель на синтетических данных

---

### Синтетические данные для RAG

Dai et al. (2022) предложили метод: имея всего 8 размеченных примеров и большой корпус неразмеченных документов, можно достичь результатов, близких к SOTA, для задачно-специфических ретриверов.

Метод: LLM генерирует запросы по документам (few-shot промпт с 2-8 примерами). Пара (синтетический запрос, документ) используется для обучения локальной модели поиска.

Стоимость: ~$55 за 50 000 документов при использовании GPT-3.5 Turbo — существенно дешевле ручной разметки.

### Промпт-функции

Промпты можно инкапсулировать в функции с именем и правилами, создавая переиспользуемые компоненты:

```
function_name: [trans_word]
input: ["text"]
rule: [I want you to act as an English translator, spelling corrector and
improver. I will provide you with input forms including "text" in any language
and you will detect the language, translate it and answer in the corrected of
my text, in English.]
```

Функции можно вызывать и комбинировать в цепочки:

```
fix_english(expand_word(trans_word('исходный текст')))
```

### Кейс: классификация вакансий

Clavié et al. (2023) применили промпт-инженерию для классификации вакансий (подходит ли для выпускника). Ключевые выводы:

- Few-shot CoT промптинг показал результаты *хуже* Zero-shot для задач, не требующих экспертных знаний
- Правильные инструкции критичны: F1 65,6 (базовый промпт) → F1 91,7 (оптимизированный)
- Присвоение модели имени повысило F1 на 0,6 пункта
- Повторение ключевых моментов (reiteration) — один из главных факторов улучшения

### Context Caching (Gemini)

Google выпустил функцию context caching для Gemini 1.5 Pro и Flash. Кэш позволяет загрузить большой контекст один раз и выполнять множество запросов без повторной отправки.

Применение: анализ сотен научных статей — загрузка резюме всех статей в кэш и интерактивные запросы: «Какие статьи упоминают Mamba?», «Какие инновации в длинноконтекстных LLM?».

---

### Дообучение GPT-4o

OpenAI предоставил возможность дообучения (fine-tuning) моделей GPT-4o и GPT-4o mini.

- Стоимость: $25 за 1M токенов обучения; $3,75/$15 за 1M токенов ввода/вывода при инференсе
- Пример: дообучение для классификации эмоций с использованием JSONL-датасета — существенное улучшение точности по сравнению со стандартными моделями
