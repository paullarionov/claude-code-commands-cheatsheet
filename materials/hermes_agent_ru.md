## Что такое Hermes Agent (за 30 секунд)

Hermes Agent набрал 50 000 звезд на GitHub за два месяца. Куча видео объясняет, что это такое. Ни одно не рассказывает, что с ним делать в первый день.

Hermes - это персональный агент автоматизации, который живет на сервере (или ноутбуке) и общается с вами через мессенджеры (Telegram, Discord, Slack, WhatsApp, Signal, Email - 15+ платформ) или через CLI. Это система, которая работает 24/7: выполняет повторяющиеся задачи, мониторит то, что вам важно, учится на своем опыте и создает переиспользуемые навыки.

Установка занимает 2 минуты:

```bash
# скачать установочный скрипт
curl ...
# запустить
hermes
# выбрать модель
hermes model
# подключить Telegram
hermes gateway setup
```

Все. У вас есть агент.

Установите его на дешевый VPS или персональное устройство. Он работает в фоне и просыпается по запросу. Пишете ему когда нужно - он помнит, что уже в работе.

## Цикл обучения (ради этого все и затевалось)

Каждые ~15 вызовов инструментов Hermes останавливается. Смотрит, что произошло. Что сработало. Что провалилось. Что заняло слишком много времени. Затем записывает навык (skill) - markdown-файл в `~/.hermes/skills/`, который превращает полученный опыт в переиспользуемый процесс.

Эти файлы не спрятаны. Их можно открыть, прочитать, отредактировать, удалить неудачные.

Практическая разница: попросите Hermes исследовать тему в первый день - получите обобщенную выжимку. Попросите то же самое на 30-й день - результат будет точнее, релевантнее и отформатирован так, как вам нравится. Он учится на том, на что вы реагируете и что игнорируете.

Память Claude Code хранит факты о ваших предпочтениях. Hermes хранит **исполняемые процедуры**. Он не просто помнит, что вам нравятся буллет-поинты. Он помнит весь процесс «исследование - фильтрация - форматирование», который выдает буллет-поинты именно так, как вы хотите.

## Чем отличается от Claude Code и OpenClaw

**Claude Code** живет в вашем репозитории. Читает код, пишет код, запускает тесты, коммитит. Один из лучших кодовых агентов. Но он не живет на сервере. Не пишет вам в Telegram. Не выполняет повторяющиеся задачи, пока вы спите.

**OpenClaw** живет на вашем сервере. Персональный агент с мессенджерами, расписанием и доступом к инструментам. Но у него нет цикла обучения. Он не пишет собственные навыки на основе опыта.

**Hermes** живет на сервере как OpenClaw, но добавляет цикл обучения. Каждая задача делает его чуть лучше в следующей. Если вы переходите с OpenClaw - одна команда импортирует все: персону, память, навыки, API-ключи, настройки мессенджеров:

```bash
hermes claw migrate
```

Пять минут - и вы перешли.

Главное: **не сравнивайте их, а используйте вместе.** Claude Code - для разработки. Hermes - для всего остального. Они используют один и тот же протокол MCP, так что ваши инструменты работают в обоих. Это и есть архитектура.

## 7 рабочих процессов

### 1. Утренние брифинги, которые учатся

Один разработчик купил Mac Mini M4 для локальных LLM. Мощности не хватило. Тогда он превратил его в домашний сервер с Hermes и Telegram-ботом. Теперь он обрабатывает пайплайн поиска работы, трекинг проектов и ежедневные сводки. Он перестал открывать ноутбук, чтобы проверить почту по утрам.

**Настройка:** подключите Hermes к Telegram через `hermes gateway setup`. Скажите мониторить почту, календарь и 2-3 темы. Поставьте на расписание. Каждое утро сводка приходит в Telegram.

Ценность не в самой сводке. Ценность в том, что происходит через две недели. Hermes узнает, на каких отправителей вы отвечаете, к каким встречам готовитесь, по каким темам задаете дополнительные вопросы. Файл навыка обновляется сам. Сводка на 30-й день не похожа на первый день.

### 2. Мониторинг веб-сайтов вместо ручного просмотра

Один разработчик настроил Hermes для проверки входящих пользовательских отчетов на работающем сайте и принятия решений о корректировке метаданных. Он полностью заменил ручной процесс ревью. Агент читает отчет, сверяет с существующими данными, применяет исправление если оно корректно и логирует изменения.

Hermes поставляется с **Camoufox** - стелс-браузером, который не палится как обычные инструменты автоматизации. Сайты, блокирующие headless-браузеры, работают чисто. Добавьте **Firecrawl** для структурированного извлечения данных - и у вас пайплайн мониторинга, который не обнаруживают.

Направьте его на страницы с ценами конкурентов, вакансии, новостные источники, каталоги товаров. Hermes занимается извлечением, отслеживанием изменений и знает, что новое, а что он уже показывал вчера. Настроили один раз - работает по расписанию.

Результат: вы перестаете вручную проверять 10 вкладок каждое утро и получаете diff того, что реально изменилось за ночь.

### 3. Один агент управляет целой компанией

Основатель финтех-стартапа попробовал мультиагентный подход. Пять специализированных ИИ-агентов: маркетинг, продажи, инженерия, комьюнити и ежедневные брифинги. Каждый со своей идентичностью, памятью и расписанием.

Провалилось за 48 часов. Агенты не могли делиться контекстом. Навыки дублировались. Голос бренда был несогласованным между каналами, потому что у каждого агента были свои инструкции.

Он свернул все в один экземпляр Hermes. Claude Code работает с кодовой базой. Один агент ведет маркетинг, аутрич, управление комьюнити и ежедневные брифинги для финтех-стартапа с нулем сотрудников. Маркетинговый контекст доступен при аутриче, потому что это один и тот же агент.

Результат: единая память означает, что каждая функция питает контекстом все остальные. Этого эффекта накопления не происходит, когда вы разбиваете работу на 5 несвязанных инструментов.

### 4. База знаний, которая умнеет со временем

Hermes поставляется с паттерном **LLM Wiki от Карпати** (Karpathy) как встроенным навыком. Вы говорите создать вики, указываете источники - он организует все в связанные markdown-файлы. Выжимки, страницы сущностей, концепций, сравнений - все с перекрестными ссылками и поддерживается агентом.

Вики имеет три слоя:

- **Сырые источники** загружаются и никогда не модифицируются
- **Вики-страницы** пишутся и поддерживаются агентом на основе источников
- **Файл схемы** определяет правила, чтобы все оставалось согласованным

Почему это важно именно для Hermes: цикл обучения означает, что вики поддерживается автоматически. Добавьте новый источник - агент не просто положит его в папку. Он проверит существующие страницы, обновит то, что изменилось, добавит перекрестные ссылки и отметит противоречия. Через месяц регулярного использования у вас база знаний с эффектом накопления, синтезирующая все, что вы в нее загрузили.

643 навыка от сообщества доступны в Skills Hub. Просмотрите их командой `/skills` внутри Hermes.

### 5. Эксперименты, которые оптимизируют себя сами

Паттерн autoresearch работает так: ИИ-агент вносит небольшое изменение, тестирует сработало ли, сохраняет победителя и пробует снова. Снова и снова. Автоматически.

Hermes создан для такого цикла. Дайте ему метрику для улучшения: open rate писем, конверсия лендинга, время ответа на лиды. Скажите вносить маленькие изменения, измерять результат и оставлять то, что работает. Цикл обучения означает, что он не тестирует случайно. Он становится лучше в предсказании того, какие изменения сработают, на основе уже попробованного.

Один разработчик дал Hermes API-ключ от брокера и создал 4 автоматизированных торговых стратегии, развернутых на реальном счете. Другой запустил автоматизированную операцию с токенами на Solana. Это не демо. Это продакшен-системы, работающие с реальными деньгами.

Результат: вы определяете цель и ограничения. Hermes проводит эксперименты, пока вы занимаетесь другим.

### 6. Подключение к MCP-серверам Claude Code (без миграции)

В Hermes v0.8.0 появилась нативная поддержка MCP-клиента. MCP - тот же протокол, который Claude Code использует для интеграции инструментов.

Каждый MCP-сервер, который вы уже собрали или установили для Claude Code, работает с Hermes. Коннектор Google Workspace, инструменты для базы данных, кастомные API. Hermes обнаруживает их автоматически. Без перестройки. Без реконфигурации.

Рабочая связка: Claude Code пишет код и управляет репозиториями. Hermes занимается исследованиями, брифингами, мониторингом и автоматизацией - используя те же MCP-серверы, которые уже настроены. Одна инвестиция в инфраструктуру, два агента, ноль дублирования.

### 7. Выбор модели (или потерянные выходные)

Неправильный выбор модели - причина номер один, почему настройки Hermes «ощущаются сломанными». Люди винят фреймворк, когда это модель не справляется с вызовами инструментов.

Один разработчик гонял Hermes на проекте почти 3 часа непрерывно после обновления v0.8.0 - но только после перехода на фронтирную модель. Другие пробовали большие open-source модели и наблюдали, как агент галлюцинирует вызовы инструментов, которых не существует.

**Gemma 4 26B через Ollama** - лучший вариант для локальных экспериментов. Но для рабочих процессов из этого гайда используйте API фронтирной модели.

Переключение:

```bash
hermes model
```

Nous Portal, OpenRouter с 200+ моделями, OpenAI или собственный эндпоинт. Без изменений кода, без привязки к вендору. Запустите `hermes doctor` если что-то не так - он диагностирует проблемы конфигурации до того, как вы потратите часы на догадки.
