## Уровень 1: основные инструменты интерфейса

Это инструменты, которыми должны пользоваться все. Самый низко висящий фрукт для ежедневной работы с Claude.

### Projects (Проекты)

Projects – самая важная функция Claude, которую большинство людей так и не настроило как следует.

Без Projects каждая беседа стартует с нуля. Claude ничего не знает о вас, вашей работе, вашем голосе и ваших стандартах – и вы тратите первые пять минут каждой сессии на повторное объяснение контекста, который уже объясняли десятки раз.

**Мой совет:** потратьте 20–30 минут, чтобы собрать файлы контекста и загрузить их в отдельные Projects. Вы сразу начнете получать лучшие ответы от AI.

### Claude Skills

Если Projects – это среда, то Skills – рабочие процессы, которые в ней выполняются.

Claude Skill – это предзагруженный набор инструкций, сохраненный как markdown-файл. Вызовите его в любом чате или Project – и Claude мгновенно начинает следовать инструкциям внутри.

Представьте так: вы собираете Skill «Brand Voice», в котором прописан ваш тон, аудитория, запрещенные слова и правила форматирования. Вместо того чтобы объяснять это каждый раз, вы говорите: «используй мой Brand Voice Skill, чтобы написать [x]».

**TL;DR как собрать:** Customize – Skills – Enable Skill-Creator. Потом говорите Claude: «я хочу собрать Skill для процесса [x], помоги мне его сделать».

Используйте Claude Skills раньше всего остального из этого гайда.

### Memory (Память)

Большинство не знает: Claude строит воспоминания о вас между беседами. Большинство позволяет этому происходить пассивно и никогда не контролирует, что именно Claude сохраняет.

Зайдите прямо сейчас в Settings – Memory и посмотрите все, что Claude о вас помнит. Удалите устаревшее или неточное и добавьте контекст, который хотите сохранить навсегда.

**Про-тип:** вы можете экспортировать память из других AI-платформ, например из ChatGPT. Просто скажите ChatGPT: «я переношу этот [проект/чат/память] в Claude, дай мне документ, который поможет с переносом».

### Connectors (Коннекторы)

Коннекторы дают Claude прямой доступ к вашим существующим инструментам внутри любой беседы.

Зайдите в Settings – Connectors и подключите те, которыми пользуетесь чаще всего. У меня это Google Drive, Gmail, Slack, Notion, Google Calendar и Excalidraw. Всего доступно 50+ коннекторов.

## Уровень 2: исследование и мышление

Ок, базу вы настроили: Skills, Connectors, Memory, Projects. Теперь начинаем использовать это для получения элитных результатов.

### Research Mode / Deep Research

Большинство знает, что эта функция есть – и игнорирует ее.

Когда вы активируете Deep Research, Claude не просто отвечает на запрос. Он разбирает его на части, ищет по десяткам источников, перекрестно сверяет находки и возвращается с развернутым отчетом со ссылками.

В зависимости от сложности вопроса это занимает от пяти до 45 минут.

Функция находится в главном окне чата, включается нажатием «+».

### Extended Thinking (Расширенное мышление)

По умолчанию Claude отвечает быстро. Для большинства задач это именно то, что нужно.

Но для сложных задач (стратегия, анализ, многошаговые рассуждения) вы хотите, чтобы Claude замедлился и подумал перед ответом.

Extended Thinking активирует более глубокое рассуждение и позволяет Claude аккуратно проработать задачу, прежде чем выдать ответ.

**Два способа включить:**

1. Через формулировку – добавьте фразы вроде «think deeply before responding»
2. Через переключатель Extended Thinking в интерфейсе чата (под выбором модели)

Помните: вы можете менять модель, с которой идет Extended Thinking. Для сложных задач с глубоким рассуждением используйте Opus 4.6.

### Artifacts (Артефакты)

Artifacts – это файлы кода (HTML-страницы, React-компоненты, документы, диаграммы, таблицы), которые существуют как отдельные файлы. Их можно просматривать, редактировать, скачивать и итеративно дорабатывать – в отличие от текста, закопанного в чате.

С Artifacts можно построить буквально что угодно, и они становятся особенно мощными в связке с кастомными Skills. Собирайте лендинги, кастомные приложения, дашборды и многое другое.

**Как использовать:** просто попросите Claude «сделай это как Artifact» или «собери это как скачиваемый файл». Artifacts включены по умолчанию.

## Уровень 3: агентские инструменты

Здесь вы начинаете раскрывать полные возможности Claude. Я пользуюсь этими инструментами ежедневно, и они помогают мне делать «настоящую работу» с Claude-агентами.

### Claude Cowork

Cowork доступен только в десктопном приложении Claude. После установки вы можете поручать Claude многошаговые задачи, которые касаются ваших реальных рабочих процессов.

Я опираюсь на три функции Cowork ежедневно:

1. **Scheduled Tasks (Запланированные задачи)** – Claude запускает задачи автоматически по расписанию. Я использую для ежедневного ресерча, сканирования коннекторов по брифам (Gmail и Calendar) и многого другого.

2. **File Access (Доступ к файлам)** – функция, которая реально отличает Cowork от других AI-инструментов. В Cowork можно дать моделям доступ к файлам и папкам на вашем компьютере. Claude может редактировать и работать внутри выделенных рабочих пространств – все локально.

3. **Plug-Ins (Плагины)** – это как Claude Skills на стероидах. Skills автоматизируют одну повторяющуюся задачу, а Plug-In упаковывает несколько Skills в единую роль.

### Cowork Dispatch

Dispatch позволяет запускать Claude Cowork с телефона.

Вы можете открыть ноутбук, оставить его работать, уйти из рабочего пространства – и при этом продолжать ставить Claude задачи. Это как мини-OpenClaw.

Просто зайдите в «Dispatch» внутри Cowork и следуйте инструкциям по настройке.

### Claude in Chrome

Этот коннектор позволяет пользователям десктопного приложения запускать задачу в Claude Desktop и выполнять ее в браузере, не переключая окон.

## Уровень 4: сборка и кодинг

Этот раздел – для тех, кто хочет собирать и кодить продвинутые приложения (не только простые Artifacts).

### Claude Code

Claude Code – самый мощный AI-инструмент для кодинга из доступных прямо сейчас.

**Что реально умеет Claude Code:**

- Разбирать сложный дебаг и ошибки в коде
- Выкатывать сайты и приложения
- Писать тесты и вести весь тестовый процесс
- Проводить аудиты безопасности
- Планировать кодинг-сессии
- Кодить на естественном языке (vibe coding)

### Slash Commands (Слэш-команды)

Встроенная система команд, которая ускоряет работу с Claude Code. Стоит сохранить шпаргалку.

**Основные команды:**

- `/help` – список всех доступных команд
- `/clear` – очистить контекст текущей сессии
- `/compact` – сжать историю разговора, освободив контекст
- `/init` – создать CLAUDE.md на основе анализа проекта
- `/memory` – открыть и редактировать файл памяти
- `/model` – переключить модель (Opus / Sonnet / Haiku)
- `/cost` – показать текущий расход токенов и денег за сессию
- `/review` – ревью pull request'а
- `/config` – открыть настройки
- `/mcp` – управлять MCP-серверами
- `/resume` – продолжить предыдущую сессию
- `/doctor` – диагностика окружения Claude Code

**Кастомные команды.** Вы можете собирать свои собственные слэш-команды. Создайте файл с именем команды и расширением `.md` в папке `.claude/commands/` внутри проекта или в `~/.claude/commands/` для глобальных. Содержимое файла – это промпт, который выполнится при вызове. Удобно для повторяющихся задач: `/deploy`, `/test-all`, `/fix-lint` и т. д.

### CLAUDE.md

Markdown-файл, который вы создаете в корне любого проекта. Claude Code автоматически читает его перед каждой сессией. Внутри – инструкции под ваш конкретный проект (стандарты кода, архитектурные решения, структура файлов, чего Claude делать нельзя).

Воспринимайте это как Project Instructions, но специально для вашей кодовой базы.

Это самое большое улучшение качества, которое большинство пользователей Claude Code не настраивают.

**Как создать:** сделайте файл `CLAUDE.md` в корне проекта и напишите все, что Claude Code должен знать, прежде чем трогать ваш код.

**Пример CLAUDE.md:**

```
Это Next.js проект с TypeScript и Tailwind.

Всегда использовать функциональные компоненты. Никогда не использовать классовые.

Запускать npm run lint перед коммитом любых изменений.

Все API-вызовы идут через папку /lib/api. Не вызывать API напрямую из компонентов.

Не модифицировать папку /config без согласования.
```

### Multi-Agent Mode / Subagents (Мульти-агентный режим)

Claude Code теперь умеет запускать несколько подагентов параллельно на разных частях одного проекта.

Например: один агент пишет тесты, пока другой собирает фичу. Один рефакторит существующий код, второй занимается документацией.

Claude Code сам решает, когда использовать подагентов – исходя из сложности задачи.

**Как их триггерить:** давайте Claude высокоуровневые промпты, ориентированные на результат, а не пошаговые инструкции.

### Memory (/memory)

У Claude Code своя система памяти, отдельная от веб-памяти. Вы можете сказать ему запомнить конкретные вещи про ваш проект, ваши предпочтения или стандарты кода. Он хранит это в файле памяти и переносит дальше.

В связке с CLAUDE.md это значит, что Claude Code постепенно накапливает понимание вашего проекта вместо того чтобы начинать с нуля каждую сессию.

Я искренне считаю: если вы внедрите эти четыре вещи – слэш-команды, грамотное ведение CLAUDE.md, триггеры подагентов и нормальное управление памятью – ваша продуктивность с Claude Code выйдет на новый уровень.
